# 安全与数据合规

## 密钥

- 只通过环境变量配置服务密钥。
- `.env.local`、真实 Token、Cookie 和供应商响应不得提交到 Git。
- `.env.example` 只能包含变量名和非敏感示例。

## 教师与教材数据

- 教师上传的教材、教案、学生信息和生成产物默认视为私有数据。
- 不得将上传文件、SQLite 开发数据库、浏览器会话或导出文件提交到公开仓库。
- 对外部模型发送内容前，应由部署方确认数据处理协议和适用授权。

## 版本与审计

- 课件修改创建新版本，不原地覆盖历史版本。
- 导出记录绑定版本哈希和 Artifact。
- 失败的视觉 QA 或审核门槛不得记录为正式可交付状态。

## 漏洞报告

请通过仓库所有者提供的私有渠道报告安全问题，不要在公开 Issue 中粘贴密钥、私人教材或可识别个人的信息。
