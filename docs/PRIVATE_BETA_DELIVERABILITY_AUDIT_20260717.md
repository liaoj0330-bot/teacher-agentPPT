# Teacher AgentPPT 真实客户链路与 12 小时收口方案

更新时间：2026-07-17 06:00（Asia/Shanghai）

## 最终裁定

当前产品不是“客户一离开开发电脑就一定乱码”，也不是“已经可以直接邀请大批老师内测”。准确状态如下：

1. 网页和 PPTX 都由部署后的 Next.js 服务端生成，老师电脑只负责访问网页、预览和下载，不承担生成计算。
2. 已有浏览器截图和 PPTX OOXML 证明中文正文是正确 Unicode；下载接口也使用 `filename*=UTF-8''...`。因此不存在“所有客户必然乱码”的结论。
3. 源码存在一处真实字符缺陷：动态扩展页使用字面量 `????`。45 分钟 9 页通常不会触发，但 60/90 分钟及扩页路径可能把问号直接送入网页和 PPTX。
4. PPTX 统一指定 `Microsoft YaHei`，但没有嵌入字体。缺少微软雅黑的电脑会字体替换，主要风险是换行、溢出和布局漂移，不是 Unicode 文字自动变成乱码。
5. 当前验收脚本固定要求 9 页，只识别少量 mojibake 特征，并依赖 Windows PowerPoint COM。它会漏掉 `????`、文件名/日志错码、动态页数和无 Office 环境，不能作为跨电脑交付证明。
6. 当前 SQLite、上传原件和解析文件依赖服务器本地磁盘；成功导出的 PPTX 字节只即时返回，Artifact 表记录的是文件名而非耐久文件。它适合带持久盘的单实例邀请制内测，不适合无状态 Serverless 或横向扩容。

因此当前裁定为：`private_beta_blocked_by_p0_deliverability`。先完成下面的 P0 收口，再邀请老师。

## 反推一节 45 分钟课还缺什么

产品输出不能只是一份学生看到的 PPT。内测最小交付包应包含：

- 学生版 PPTX：可编辑、可投影、题目与答案分离、无占位符。
- 教师版课堂脚本：每个教学事件的分钟数、教师动作、学生动作、追问、预期回答和失控时的备用动作。
- 课堂任务与答案：练习题、实验/阅读任务、标准答案、评分点、常见错误和二次反馈。
- 板书设计：板书骨架、出现顺序和最终留存内容。
- 分层作业：基础、提高、挑战及评价标准。
- 课前 QA：教材对齐、事实正确、总时长、活动闭环、答案完整、字体和导出完整性。

45 分钟规划必须从“页数”升级为“教学事件”。推荐合同：

```text
LessonPlan
  -> 课程目标与教材边界
  -> LessonEvent[]
       durationMinutes
       teacherAction
       studentAction
       expectedResponse
       evidenceOfLearning
       fallbackAction
       slideIds[]
  -> 总时长 45 分钟校验
  -> StudentDeck + TeacherNotes + AnswerKey + BoardPlan + Homework
```

页数由事件和内容密度推导，不把 45 分钟硬编码为 9 页。物理实验课可以 13-16 页，语文精读课可以 11-15 页；关键是教学事件和时间闭环，而不是页数越多越好。

## 100 个客户场景推演

本轮使用 10 组、每组 10 个场景进行源码、既有产物和可运行回归的组合审计。它不是 100 次付费模型调用；没有真实执行证据的场景统一标记为未验证。

| 编号 | 10 个场景 | 结论 |
|---|---|---|
| A01-A10 进入与身份 | 新用户、老用户、无邀请码、会话过期、桌面、手机、刷新、后退、重复登录、多标签 | 基础 UI 通过；会话异常恢复未全验 |
| B01-B10 教材输入 | 章节、教案、PDF、Word、粘贴、旧 PPT、缺字段、错版本、超大文件、损坏文件 | 三入口存在；格式与失败矩阵未闭环 |
| C01-C10 课时策划 | 20/25/30/40/45/50/60/90/120 分钟、教师自定义扩页 | 7/9/10/12/14 页逻辑存在；60 分钟以上可触发 `????`，失败 |
| D01-D10 学科学段 | 小学数学、初中数学、小学语文、初中语文、小学英语、初中英语、初中物理、高中物理、初中化学、高中化学 | 数/语/英部分验证；物理化学深度未验证 |
| E01-E10 课堂完整性 | 目标、导入、讲解、实验/阅读、互动、练习、答案、纠错、板书、作业 | 字段零散存在；没有统一 45 分钟交付合同，失败 |
| F01-F10 编辑版本 | 改标题、改正文、删页、加页、换序、单页重做、整套重做、补材料、回退、并发冲突 | API/版本模型存在；连续 10 次真实 E2E 未通过 |
| G01-G10 导出字符 | 纯中文、中文文件名、中英混排、公式、标点、生僻字、emoji、`????`、GBK 上传、UTF-8 上传 | Unicode 主体正确；问号、字体和编码门禁不完整 |
| H01-H10 跨电脑打开 | Windows+Office、Windows 无 Office、macOS Office、WPS、LibreOffice、缺微软雅黑、浏览器下载、重复下载、断网重开、移动下载 | 仅 Windows PowerPoint 历史证据；其他环境未验证 |
| I01-I10 故障恢复 | 模型超时、接口 429、余额不足、网络中断、浏览器关闭、服务重启、磁盘满、DB 锁、chunk 失败、重复请求 | 部分错误可见；长任务和部署恢复不完整 |
| J01-J10 运营交付 | 配额、费用、日志、监控、备份、恢复、数据删除、隐私、教师反馈、第二次使用 | 主要仍是设计项，未达到成熟运营 |

100 个场景的总裁定不是“100/100 通过”，而是暴露了 6 个 P0：动态扩页问号、课堂编排缺失、答案/教材事实占位、字体与跨环境门禁、产物非耐久、云端单实例未部署。任何状态卡不得再把动态页数或单份 PPTX 结构通过写成完整交付通过。

### 18:00 可以承诺的程度

若模型文本接口、现有数据库和本机服务稳定，12 小时目标是“邀请制内测 RC1 候选”，不是成熟商用：

- 无图片主链可交付，不因图片供应商阻塞。
- 45 分钟不再只是机械 8-9 页，具备事件时间、教师动作和学生产出。
- 高二物理与初中语文两条产品真实链有可复现证据。
- 动态扩页不出现 `????`，字符/占位/页数门禁自动化。
- 下载、重开、再次下载和 10 次连续修改有报告。
- 仍需真实教师人工过课，以及实际云服务器、域名和 HTTPS 凭据才能对外开放。

### 12 小时不能诚实承诺的事情

- 所有学科、所有教材和所有课型都成熟。
- PostgreSQL、对象存储、队列、Worker、多实例和完整运营后台全部上线。
- macOS、WPS、LibreOffice、所有 Office 版本都完成真实渲染认证。
- 图片长任务稳定化；本轮明确排除图片模型。
- 老师无需审核即可把任何 AI 内容直接带进课堂。

## 成熟产品目标与工期

成熟链路应为：认证与配额 -> 教材/材料事实链 -> 课堂事件策划 -> 教师确认 -> 页面生成 -> 无图/图片资产任务 -> 可编辑预览 -> 版本提交 -> 服务端导出 -> 跨环境渲染 QA -> 耐久下载 -> 课堂反馈与复用。

- 12 小时：完成 P0 收口，目标为 RC1 候选。
- 36-48 个工程小时：完成单实例云端邀请制内测、部署恢复、核心五学科矩阵和跨环境抽样。
- 80-120 个工程小时：完成 PostgreSQL、对象存储、后台任务、可观测性和多用户稳定性，达到可扩大内测的工程成熟度。
- 真实成熟仍需要至少一轮 5-10 位教师、每人一节真实课的封闭验证；工程通过不能替代教师验收。

## 放行门槛

只有同时满足以下条件，状态才能从 `private_beta_blocked_by_p0_deliverability` 改为 `private_beta_rc1`：

1. 源码、网页、PPTX、文件名和报告五层字符扫描通过。
2. 25/45/60/90 分钟规划无问号、无固定页数假象、总时长一致。
3. 两门真实课的主入口 E2E 均完成生成、修改、保存、导出和重开。
4. 教师脚本、答案、板书和作业都有实际内容，无“教师补充”占位符。
5. 无图片模式不调用图片 API 且可以完成全部交付。
6. 导出文件可再次下载，版本与 Artifact 哈希一致。
7. 连续 10 次修改和版本回退通过。
8. 至少一套 PowerPoint/WPS 真实打开截图与人工过课记录完成。
