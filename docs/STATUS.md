## 已验证事实

- 唯一主入口为 `/teacher-ai-ppt`；旧入口组件已删除。
- TypeScript 检查与 Next.js 生产构建通过。
- 版本事实链真实 HTTP E2E 10/10。
- 13 类教师 LayoutContract、RenderScene、Page Gate、Visual QA 已建立。
- 浏览器预览与 PPTX 导出共享统一场景模型。
- 原生文本、形状、表格和图表保持可编辑。
- 真实无图片流程生成 9 页、保存版本并导出 PPTX，总耗时约 1 分 47 秒。
- 历史 PowerPoint 环境曾对 9 页样例真实渲染 9/9；本轮环境缺少 PowerPoint COM，只完成 OOXML 结构与 Unicode 检查，不能外推为跨电脑通过。
- 数学、语文、英语的角色、标题和可见内容能够随学科变化，未观察到已知跨学科泄漏。
- 课前评分 V3 覆盖教材对齐、教学设计、学科正确、视觉表达、工程质量和教师效率。
- 原始教材/教案文件已进入耐久 SourceAsset 事实链。

## 图片链现状

已完成：

- 图片供应商已单一化为 OpenAI-compatible 通道。
- 当前模型合同为 `gpt-image-2`、SSE、全局并发上限 3。
- 失败返回 HTTP 502，不再使用本地 SVG 冒充远程成功。
- 3 路并发真实验收曾达到 3/3 成功。
- 已有 31 张真实成功图片记录，单张约 34～123 秒。
- 0/N 图片允许使用原生可编辑视觉；N/N 图片允许整套提交；部分图片成功会被交付策略阻断。

未完成：

- 当前供应商余额不足，模型列表可读，但真实出图返回 insufficient balance。
- 页面级异步任务、即时持久化和断点续跑尚未建立。
- 历史完整图片流程出现过重复重试、长时间等待、超时和额度快速消耗。
- 图片版本资产、浏览器场景和 PPTX 场景的整套同源验收仍需恢复后重跑。

因此当前只能标记为 `image_provider_partially_verified`，不能标记为稳定生产链。

## 跨学科边界

当前输入层支持幼儿园、小学、初中、高中、中职、大学，以及语文、数学、英语、物理、化学、生物、历史、地理等学科。

已验证到内容差异的学科：数学、语文、英语。

仍需真实教材验收的学科：物理、化学、生物、历史、地理，以及高中和中职的复杂课程。

目前物理、化学等可以进入通用理科策划和概念课模板，但尚无足够证据证明实验设计、变量控制、现象观察、公式推导、误差分析等学科动作能够稳定变化。

## 教师调整与响应

产品已经具备大纲确认、对话调整、补充材料、创建新版本、读取历史版本和重新导出能力。

尚待验证：

- 连续 10 次修改不丢版本、不改错页、不重复扣费。
- 局部文字调整、AI 内容调整、整页重做和图片重生成的响应时间分层。
- 网络中断或浏览器关闭后，长任务能否恢复。
- 不同教师对“保留原稿”和“深度重构”的偏好是否被稳定执行。

## 部署状态

当前使用 SQLite 和本地文件路径，适合本地验证或单机小规模测试，不适合直接横向扩容。

邀请制内测建议先使用一台带持久磁盘的云服务器：Next.js 应用、SQLite、上传文件和课件产物保持在同一受控实例，通过 HTTPS 域名提供访问。教师只打开 URL，不接触模型 API Key。

扩大规模前需要迁移到 PostgreSQL、对象存储和后台任务队列，并补齐备份、日志、监控、配额和数据删除能力。
