# 教师课件评分与验收分层

## 当前基线

评分必须区分机器可验证事实和真实课堂证据，不能把一次工作流按钮提交解释成完成试讲。

| 层级 | 当前分值 | 证据来源 | 能证明什么 | 不能证明什么 |
| --- | ---: | --- | --- | --- |
| 自动检查 | 0-95 | 教材身份、逐页证据、教学模块、渲染与 OOXML 检查 | 课件结构、内容和文件是否达到课前复核线 | 教师是否能顺畅讲完、学生是否理解 |
| 教师流程确认 | 不计分 | `teacher_submit_for_review` 产生的版本状态 | 有人执行了审核提交动作 | 真实 45 分钟试讲已经发生 |
| 真实教师试讲 | 0 或 5 | 教师身份、试讲时间、量表回答、问题记录、学生反馈 | 课件是否能直接上课 | 跨学校、跨设备普遍可用 |
| 商业发布门禁 | 不计分，硬门禁 | WPS/PowerPoint 目标机、真实教师试讲、云端部署等外部证据 | 是否可以对外声明可交付 | 不能由高分绕过 |

当前 V3 自动检查权重为：教材对齐 25、教学设计 25、学科正确 20、视觉表达 15、工程质量 10，总计 95。只有结构化真实试讲证据占 5 分；流程提交不计分。

## 发布判定

- `automaticScore < 80/95`：不进入教师试讲。
- 存在 P0：禁止教师正式版导出；修复后重新评分。
- `automaticScore >= 80/95` 且无 P0：允许生成教师复核稿。
- 教师流程确认完成：只表示版本进入教师审核通过状态，不自动生成试讲结论。
- 真实试讲资料不完整：`classroomTrial.status=pending`，不得声明课堂验证完成。
- 任一 release gate 未关闭：`commercialReady` 必须为 `false`，分数不得覆盖硬门禁。

## 真实试讲最小量表（待产品化）

每次试讲至少记录以下字段，缺一项就不能标记为完成：

1. 教师与班级：匿名教师 ID、学段、年级、学科、班级规模。
2. 试讲事实：开始/结束时间、计划时长、实际时长、使用的软件和设备。
3. 教学执行：目标达成、节奏、互动、练习反馈、讲稿可用性，各 1-5 分并附说明。
4. 异常记录：乱码、溢出、图片不匹配、知识错误、跳页和临时改动。
5. 结果确认：教师是否愿意原样再用、必须修改项、签名确认时间。

## 当前量化盲点

- 真实试讲结构化记录：已具备证据合同并随不可变版本保存，尚缺前端量表录入界面。
- 教师验证分辨率：2 档（0/5），无法表达“能上但需修改”。
- 学生学习效果：0 项指标。
- WPS/PowerPoint 跨设备评分：尚未进入 V3 分值，仅能作为外部硬门禁。
- 学科人工复核和图片语义复核：接口接受布尔值与问题数，但没有复核人和证据附件。

运行 `npm run teacher-scoring-provenance:test` 生成定量审计结果。该测试不调用图片模型。
