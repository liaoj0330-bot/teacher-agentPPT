# Teacher AgentPPT 内测准备路线图

更新时间：2026-07-16

## 目标

把当前“可用产品候选”推进为可供真实教师小范围体验的邀请制内测版本。重点不是继续堆功能，而是证明不同学科、不同材料、不同教师修改方式和不同网络状态下都能得到可预测结果。

## 阶段 1：先把非图片主链做扎实

### 1.1 跨学科策划矩阵

最低案例：

| 学科 | 学段案例 | 必须变化的教学动作 |
|---|---|---|
| 数学 | 小学、初中 | 操作、数形对应、例题步骤、纠错 |
| 语文 | 小学、初中 | 朗读、词句理解、段落结构、表达迁移 |
| 英语 | 小学、初中 | 听说输入、情境对话、语言输出 |
| 物理 | 初中、高中 | 现象、实验、变量、规律、计算与误差 |
| 化学 | 初中、高中 | 物质、实验装置、现象、反应与安全 |
| 生物 | 初中、高中 | 结构、过程、观察、实验与证据 |
| 历史 | 初中、高中 | 时序、史料、因果、比较与评价 |
| 地理 | 初中、高中 | 地图、空间关系、过程、数据与区域分析 |

每个案例必须使用真实教材章节或教师材料，检查页面角色、内容指纹、学科禁用词、事实正确性和课堂动作，不能只检查标题变化。

### 1.2 三条主线 E2E

- 教材章节：从教材身份到最终 PPTX。
- 教案材料：原始文件、解析、引用、生成和版本保存。
- 优化旧课件：三档强度、原稿保真、结构差异和可编辑导出。

### 1.3 教师连续修改

模拟不同教师要求：

- “内容别动，只换版式。”
- “删掉两页，增加一个课堂活动。”
- “这页太难，改成初一学生能懂的表达。”
- “只重做第 4 页，其他页保持不变。”
- “补充一段教材，再生成新版本。”

验收连续 10 次操作，确认旧版本不变、目标页正确、页面 ID 稳定、导出可用、失败可恢复。

## 阶段 2：建立时间与体验预算

目标口径：

| 动作 | 内测目标 |
|---|---|
| 大纲预览 | 10 秒内 |
| 无图片完整课件 | P50 2 分钟内 |
| 本地文字/顺序修改 | 2 秒内可见 |
| AI 内容调整 | P50 60 秒内 |
| 单页重做 | P50 90 秒内 |
| 4 张图片课件 | P50 5 分钟内，P90 8 分钟内 |

每个长动作必须展示当前阶段、完成页数、失败页和可重试动作，不能只显示无限加载。

## 阶段 3：恢复并收口图片链

恢复余额后按以下顺序执行：

1. 单张低质量探针，确认鉴权、模型、SSE、图片签名和费用。
2. 设置本轮预算和最大成功图片数。
3. 3 路并发小批次，不运行整套课件。
4. 建立 `versionId + slideId + promptHash` 幂等键。
5. 页面完成即持久化，失败只重试当前页。
6. 显式记录 `remote_success`、`native_fallback`、`failed`。
7. 验证同一资产进入浏览器和 PPTX。
8. 最后才运行一次完整教师流程。

禁止再次以“多跑几次看稳不稳”的方式消耗整套图片额度。

## 阶段 4：邀请制云端内测

### 第一阶段部署

- 单台云服务器或单容器实例。
- 持久磁盘保存 SQLite、上传文件和导出产物。
- 域名、HTTPS、邀请码登录。
- 环境变量保存在服务器，前端永不暴露供应商密钥。
- 每日数据库与文件备份。
- 请求日志、错误日志、任务耗时和费用日志。
- 用户级生成次数、图片张数和并发上限。

### 扩大用户前的升级

- SQLite -> PostgreSQL。
- 本地文件 -> 对象存储。
- 同步长请求 -> 后台任务队列和 Worker。
- 单实例 -> 多实例与共享状态。
- 增加数据导出、删除和保留期限策略。

## 阶段 5：真实教师验收

建议首批 5～10 位教师，每位完成一节真实课程，记录：

- 从开始到得到可讲课件用了多久。
- 哪些页面可以直接使用，哪些必须修改。
- 教材是否对齐，事实是否正确。
- 修改是否比自己重做更省时间。
- 是否愿意第二次主动使用。

内测成功指标不是“生成成功率”单项，而是：完成率、首次可用时间、教师修改次数、导出成功率、第二次使用意愿和问题恢复率。
